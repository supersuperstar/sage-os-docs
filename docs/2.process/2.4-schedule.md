# 2.4 进程调度
## 概述
进程调度的主要功能是按照一定的策略选择—个处于**就绪状态**的进程，使其获得cpu并执行。
## 接口说明
```c
/**
 * @brief handler: schedule next task to run
 *        Notice: this should be the only handler which returns real Context
 *
 * @param ev
 * @param context
 * @return Context*
 */
Context *kmt_schedule(Event ev, Context *context);
```
